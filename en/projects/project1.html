About the Map

The map presents spatial data that allows assessing population distribution, existing food service infrastructure, and calculating an analytical indicator — the F&B market saturation index. The map is intended for preliminary location analysis when planning the opening of new food outlets, as well as for studying the competitive environment in various urban districts.


Population Data

Population information is presented in the form of hexagonal cells (H3). This format is convenient for spatial analysis because all cells have the same area and do not distort the real distribution of people, unlike administrative boundaries. The data is obtained from the open source Kontur Population, which aggregates information from various satellite and statistical data. The hexagon color varies from light pink to dark burgundy depending on the number of inhabitants: the darker the color, the more people live in that cell.


Food Outlets

Existing food service establishments from the open cartographic source OpenStreetMap are plotted on the map. The selection includes the following categories:

  • restaurants
  • cafes
  • bakeries
  • bars and pubs
  • fast food establishments
  • food courts

Each point has its own color depending on the type of establishment. This allows you to visually assess which formats predominate in a particular district and identify possible niches not occupied by competitors.


F&B Market Saturation Index

For each hexagon, the F&B market saturation index is calculated. This indicator is calculated as the ratio of the number of residents to the number of food outlets within a single hexagon.

Formula: Saturation Index = Hexagon Population / Number of Food Outlets in Hexagon


Interpretation of the Index:

  • Low values (less than 500 residents per food outlet) — high concentration of establishments, market oversaturation, high competition.
  • Medium values (500–1500 residents per food outlet) — moderate competition, balanced market.
  • High values (more than 1500 residents per food outlet) — lack of supply, presence of a free niche for opening a new business.

The color scheme of the index is built on a gradient from red (high competition) to green (free niche).


Interactive Features

The map supports interaction to obtain detailed information about the selected location.

  • Click on the map — automatically builds a 5-minute isochrone (pedestrian accessibility zone). The isochrone shows the territory that a person can walk in 5 minutes from the selected point (at an average pedestrian speed of 80 meters per minute, the radius is 400 meters). Inside this zone, the number of existing food outlets (competitors) is counted. The result is displayed in a pop-up window.

Statistics Panel

A panel is located at the top of the screen, which displays in real time:

  • population of the selected hexagon
  • population within the isochrone zone

Layer Switching

Using the control buttons, you can enable and disable the display of layers:

  • Population layer — displays hexagons with color indication of population density.
  • Market saturation index layer — shows the estimated number of potential customers for food outlets (an aggregated indicator based on population density and other factors).
  • Reset isochrone — The button allows you to delete the isochrone created by the user.

Technologies and Data Sources

  • Mapping library: MapLibre GL JS
  • Spatial analysis: Turf.js (calculation of isochrones, intersections, buffer zones)
  • Population data: Kontur Population (hexagons)
  • Food outlet data: OpenStreetMap (OSM)
  • Map style: OpenFreeMap / Positron

I implemented this project on the frontend; the data was loaded as geojson files directly into the project. This approach is due to the technical limitations of GitPages. If necessary and if technical capabilities are available, it is possible to implement the project with a PostgreSQL database deployment and/or connection of a vector tile service.
